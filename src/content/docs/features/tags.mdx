hiaryAI의 태그 시스템은 학습 기록을 체계적으로 분류하고 검색할 수 있도록 도와주는 기능입니다.

AI가 글 내용을 분석해서 관련 태그를 자동으로 제안하고, 개인 맞춤 분류 체계를 구축할 수 있어요.

## 태그 시스템 개요

### 태그의 역할
- **분류**: 학습 내용을 주제별로 체계적으로 정리
- **검색**: 특정 주제의 글들을 빠르게 찾기
- **추천**: 비슷한 관심사의 다른 학습자들과 연결
- **분석**: 자신의 학습 패턴과 관심 분야 파악

### 태그 유형
- **주제 태그**: 학습 분야나 기술 스택 (#React, #JavaScript, #영어문법)
- **난이도 태그**: 학습 수준 표시 (#기초, #중급, #고급)
- **유형 태그**: 콘텐츠 성격 (#개념정리, #실습, #프로젝트, #트러블슈팅)
- **목표 태그**: 학습 목적 (#취업준비, #자격증, #개인프로젝트)

## AI 자동 태깅

### 스마트 태그 제안

**내용 분석 기반 제안**
- AI가 글의 내용을 분석해서 관련성 높은 태그들을 자동으로 제안해요
- 제목, 본문, 코드 블록까지 종합적으로 분석
- 사용자의 이전 태깅 패턴을 학습해서 개인화된 제안

**예시:**

🏷️ **[이미지 플레이스홀더: AI 태그 제안 화면]**

```
작성한 글: "React Hook useState 사용법과 주의사항"

AI 제안 태그:
- #React (주제 - 확실성 95%)
- #Hook (주제 - 확실성 90%)  
- #useState (세부기술 - 확실성 88%)
- #프론트엔드 (분야 - 확실성 85%)
- #중급 (난이도 - 확실성 70%)
```

### 태그 신뢰도 시스템
- **확실성 점수**: AI가 제안하는 각 태그의 확실성을 0-100% 로 표시
- **학습 향상**: 사용자가 태그를 수정하면 AI가 학습해서 다음에 더 정확한 제안
- **개인화**: 개인의 태깅 스타일과 선호도를 점진적으로 학습

## 태그 관리 시스템

### 개인 태그 라이브러리

**내 태그 페이지**

📚 **[이미지 플레이스홀더: 개인 태그 라이브러리]**

- 지금까지 사용한 모든 태그들의 목록
- 각 태그별 사용 빈도와 최근 사용일
- 태그별로 분류된 글 목록 바로가기

**태그 통계**
- 가장 많이 사용한 태그 Top 10
- 최근 관심 분야 변화 추이
- 태그 사용 패턴 시각화

### 태그 편집과 관리

**태그 편집**
- **이름 변경**: 기존 태그 이름을 새로운 이름으로 변경
- **태그 병합**: 비슷한 의미의 태그들을 하나로 합치기
- **태그 분할**: 너무 포괄적인 태그를 세분화하기

**예시 시나리오:**
```
문제: #js 와 #JavaScript 태그가 혼재
해결: "태그 병합" 기능으로 #JavaScript로 통일

문제: #웹개발 태그가 너무 포괄적
해결: #프론트엔드, #백엔드, #풀스택으로 분할
```

### 태그 계층 구조

**상위-하위 태그 관계**
```
#프로그래밍
  ├── #웹개발
  │   ├── #프론트엔드
  │   │   ├── #React
  │   │   ├── #Vue
  │   │   └── #Angular
  │   └── #백엔드
  │       ├── #Node.js
  │       ├── #Python
  │       └── #Java
  └── #모바일개발
      ├── #iOS
      └── #Android
```

**계층적 필터링**
- 상위 태그 선택 시 하위 태그 글들도 함께 표시
- 세분화된 검색과 포괄적 검색을 자유롭게 전환

## 태그 기반 탐색

### 개인 피드 커스터마이징

**관심 태그 팔로우**
- 자주 보고 싶은 태그들을 팔로우
- 팔로우한 태그의 새 글들이 피드에 우선 표시
- 태그별 알림 설정으로 새 글 업로드 시 알림 수신

**태그 조합 필터**
```
예시 조합:
- #React AND #초급 : React 초급 글만 보기
- #영어 OR #중국어 : 언어 학습 관련 글들
- #프로젝트 NOT #취업 : 개인 프로젝트 글 (취업용 제외)
```

### 태그별 커뮤니티

**태그 전용 페이지**
- 각 태그별로 전용 커뮤니티 공간
- 해당 태그의 인기 글, 최신 글, 우수 글 모음
- 같은 태그를 사용하는 활발한 사용자들 추천

**태그 리더보드**
- 특정 태그에서 가장 활발한 사용자 순위
- 가장 도움이 되는 글을 많이 작성한 사용자
- 태그별 전문가 인증 시스템

## 태그 활용 전략

### 효과적인 태깅 방법

**태그 개수 최적화**
- **권장 개수**: 글당 3-7개 태그
- **너무 많으면**: 분산되어 검색 효과 떨어짐
- **너무 적으면**: 발견되기 어려워짐

**태그 선택 기준**
1. **주제 태그**: 가장 핵심적인 주제 1-2개
2. **세부 태그**: 구체적인 기술이나 개념 2-3개  
3. **난이도 태그**: 대상 독자 수준 1개
4. **유형 태그**: 글의 성격 1개

**좋은 태깅 예시:**
```
글: "React Hook을 사용한 간단한 투두리스트 만들기"

추천 태그:
#React (주제)
#Hook (세부기술)  
#투두리스트 (프로젝트)
#초급 (난이도)
#실습 (유형)
```

### 개인별 태깅 전략

**개발자용 태깅 체계**
- **기술 스택**: #React, #Node.js, #MongoDB
- **개발 영역**: #프론트엔드, #백엔드, #데브옵스
- **학습 단계**: #기초, #실무, #심화
- **프로젝트 성격**: #토이프로젝트, #포트폴리오, #실무

**언어 학습자용 태깅 체계**
- **언어**: #영어, #일본어, #중국어
- **스킬**: #듣기, #말하기, #읽기, #쓰기
- **수준**: #초급, #중급, #고급
- **목적**: #여행, #비즈니스, #시험

## 태그 분석과 인사이트

### 학습 패턴 분석

**태그 사용 통계**
- 월별 태그 사용 변화 추이
- 가장 성장한 분야 (태그 사용 증가율)
- 관심 분야의 다양성 지수

**학습 균형 체크**
- 이론 vs 실습 비율 (#개념정리 vs #실습)
- 깊이 vs 넓이 (#심화 vs #입문)
- 개인 vs 협업 (#개인프로젝트 vs #팀프로젝트)

### AI 기반 추천

**다음 학습 추천**
- 현재 태그 패턴을 분석해서 다음에 공부하면 좋을 주제 추천
- 부족한 영역 식별 및 보완 방향 제시
- 비슷한 패턴의 다른 학습자들이 성공한 경로 제안

**태그 최적화 제안**
- 사용하지 않는 태그 정리 제안
- 더 효과적인 태그 조합 추천
- 트렌드에 맞는 새로운 태그 제안

## 태그와 다른 기능 연동

### 복습 시스템 연동

**태그별 복습 스케줄**
- 중요한 태그는 더 자주 복습 알림
- 어려운 태그는 더 짧은 주기로 반복
- 개인의 태그별 이해도에 따른 맞춤 복습

### 시리즈 연동

**태그 기반 시리즈 제안**
- 같은 태그의 글들을 자동으로 시리즈로 묶기 제안
- 연관성 높은 태그들의 글을 순차적 학습 경로로 제안

## 커뮤니티 태그 문화

### 태그 에티켓

**좋은 태깅 관례**
- **정확한 태그**: 내용과 일치하는 태그만 사용
- **표준 태그**: 커뮤니티에서 일반적으로 사용하는 태그 활용
- **적절한 개수**: 과도한 태그 남용 지양

**피해야 할 태깅**
- **태그 스팸**: 관련 없는 인기 태그 남발
- **모호한 태그**: #좋음, #어려움 같은 주관적 태그
- **중복 태그**: #React, #ReactJS 같은 의미 중복

### 태그 트렌드

**인기 태그 추이**
- 실시간 인기 태그 랭킹
- 급상승하는 기술/주제 태그
- 계절별, 시기별 태그 트렌드 분석
