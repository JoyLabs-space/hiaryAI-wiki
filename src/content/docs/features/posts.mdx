게시물은 매일의 학습 내용을 간단하고 체계적으로 기록할 수 있는 기능입니다.

긴 글 형태의 학습 기록과는 달리, 짧고 집중적인 내용을 빠르게 정리할 수 있도록 설계되었어요. 지속적인 기록 습관을 형성하고 일일 학습을 축적할 수 있습니다.

## 게시물의 특징

### 간단한 기록 방식
- **빠른 작성**: 복잡한 서식 없이 핵심 내용만 간단히 작성
- **일일 단위**: 하루에 학습한 내용을 한 번에 정리
- **템플릿 지원**: 미리 준비된 템플릿으로 일관된 기록 가능

### 자동 관리 기능
- **날짜별 분류**: 작성일을 기준으로 자동 정리
- **검색 기능**: 키워드 검색으로 빠른 내용 찾기

## 게시물 작성하기

### 새 게시물 만들기

**1. 게시물 작성 시작**

📝 **[이미지 플레이스홀더: 게시물 작성 시작하기]**

- 대시보드에서 "새 게시물" 버튼 클릭
- 또는 사이드바의 "게시물" 섹션에서 "+" 버튼

**2. 날짜 선택**
- 기본적으로 오늘 날짜가 선택됨
- 필요시 다른 날짜 선택 가능 (과거 날짜 보충 기록용)

**3. 기본 템플릿**
```
# 2024년 12월 19일 게시물

## 오늘 학습한 것
-

## 새로 알게 된 점
-

## 어려웠던 점
-

## 내일 학습할 것
-
```

### 효과적인 게시물 작성법

**핵심 원칙**
- **구체적으로**: "React 공부했다" (X) → "React Hook useState 사용법 익혔다" (O)
- **간결하게**: 각 항목마다 1-2줄로 핵심만 정리
- **일관되게**: 매일 같은 형식으로 작성하여 습관 형성

**작성 팁**
- **코드 스니펫 포함**: 중요한 코드는 간단한 예시로 포함
- **참고 자료 링크**: 도움이 된 자료 URL 첨부
- **감정 기록**: 학습하면서 느낀 점도 간단히 메모

## 게시물 관리하기

### 날짜별 보기

📅 **[이미지 플레이스홀더: 게시물 캘린더 뷰]**

- **캘린더 뷰**: 월별 캘린더에서 게시물 작성 현황 한눈에 확인
- **연속 작성일**: 몇 일 연속으로 게시물을 작성했는지 표시
- **누락일 표시**: 작성하지 않은 날짜를 시각적으로 표시

### 주제별 분류
- **내용 기반 필터링**: 특정 주제의 게시물만 모아서 보기

### 검색과 필터
- **키워드 검색**: 전체 게시물에서 특정 내용 검색
- **날짜 범위**: 특정 기간의 게시물만 조회
- **고급 필터링**: 여러 조건을 조합한 필터링

## 게시물과 복습 시스템 연동

### 자동 복습 생성
- **게시물 내용 기반**: 작성한 게시물을 바탕으로 AI가 복습 질문 생성 *(개발 예정)*
- **주기적 알림**: 망각곡선에 따른 게시물 복습 알림
- **패턴 분석**: 자주 다루는 주제는 더 자주 복습 제안

### 학습 패턴 분석
- **일일 학습량**: 매일의 학습 분량 시각화
- **주제 분포**: 어떤 분야를 얼마나 학습하는지 분석
- **성장 추이**: 시간에 따른 학습 깊이 변화 추적

## 커뮤니티와 게시물

### 다른 사용자 게시물 구경
- **게시물 피드**: 공개된 다른 사용자들의 게시물 구경
- **영감 받기**: 비슷한 분야 학습자들의 학습 방법 참고
- **스터디 메이트**: 같은 주제를 공부하는 사람들과 연결

## 게시물 활용 시나리오

### 개발자의 하루

**오전 (새 기술 학습)**
```
# 2024-12-19 게시물

## 오늘 학습한 것
- Next.js 13 App Router 구조 파악
- Server Components vs Client Components 차이점

## 새로 알게 된 점
- App Router에서는 기본적으로 모든 컴포넌트가 Server Component
- 'use client' 지시어로만 Client Component 지정 가능

## 실습 코드

```javascript
// app/page.tsx (Server Component)
export default function HomePage() {
  return <div>Hello App Router!</div>
}

// components/Counter.tsx (Client Component)
'use client'
import { useState } from 'react'
export default function Counter() {
  const [count, setCount] = useState(0)
  return <button onClick={() => setCount(count + 1)}>{count}</button>
}


## 어려웠던 점
- 언제 Server Component를 쓰고 언제 Client Component를 써야 하는지 혼란스러움

## 내일 학습할 것
- Next.js 13 라우팅 시스템 자세히 알아보기
- 실제 프로젝트에 App Router 적용해보기
```

### 언어 학습자의 기록

**저녁 (영어 회화 수업 후)**
```
# 2024-12-19 영어 학습 게시물

## 새로운 단어/표현
- "I'm swamped" = 매우 바쁘다
- "Let me get back to you" = 나중에 연락드릴게요
- "That rings a bell" = 어디서 들어본 것 같다

## 문법 포인트
- 현재완료 vs 과거시제 차이
- "I have been to Seoul" (경험) vs "I went to Seoul yesterday" (과거사실)

## 실수한 부분
- "I have went" (X) → "I have been" (O)
- 과거분사 형태 헷갈림

## 실습한 회화
- 상황: 친구와 여행 계획 세우기
- 사용한 표현: "Have you ever been to...?", "I'd love to visit..."

## 다음 학습 목표
- 현재완료 시제 하기
- 여행 관련 어휘 더 늘리기
```

## 게시물 성공 비법

### 습관 만들기
- **고정 시간**: 매일 같은 시간에 게시물 작성 (예: 오후 9시)
- **작은 것부터**:  글보다는 짧아도 꾸준한 기록
- **연속 기록**: 하루도 빠뜨리지 않고 연속 기록에 도전

### 동기 부여 유지
- **성취감 시각화**: 달성한 연속일수를 시각적으로 확인
- **성장 추적**: 과거 게시물과 비교해서 성장한 모습 확인
- **커뮤니티 참여**: 다른 사람들과 게시물 공유하고 격려받기
